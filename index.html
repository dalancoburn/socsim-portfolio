Case Studies
BEC-2024-001

Business Email Compromise — CFO Account Takeover

AiTM phishing kit captured the CFO's session token post-MFA, granting adversary persistent Exchange access. A forwarding rule silently exfiltrated 14 emails before a fraudulent wire transfer of $127K was identified and blocked. Total dwell time: 6 hours. Detection-to-containment: 58 minutes.

Completed in a
live SOCSIM lab
Alert Trigger
Microsoft Sentinel — High Severity "Impossible Travel: Sign-in from multiple geographic locations within 4-hour window" triggered at 02:14 UTC for cfo@[org].com. Sentinel fusion appended a co-occurring "MFA Fatigue: 7 consecutive push-notification denials" alert from the same session, escalating both indicators into a single P1 incident.
Methodology
Preparation → Identification → Containment → Eradication → Recovery → Lessons Learned

Root Cause
An AiTM (Adversary-in-the-Middle) phishing kit hosted at credential-secure-login[.]com operated as a reverse proxy to the legitimate Microsoft sign-in portal. After the CFO completed MFA, the kit captured the post-authentication session cookie and replayed it from a Tor exit node. Conditional Access did not re-evaluate device compliance on IP change. Legacy Continuous Access Evaluation (CAE) enforcement was absent from executive accounts, allowing the stolen token a 6-hour unrestricted access window.
Containment
All active sessions revoked via Microsoft Graph API (revokeSignInSessions), immediately invalidating the stolen refresh token. Conditional Access updated to enforce CAE strict-location mode for all C-suite accounts. Phishing domain submitted to Microsoft DART for tenant-level block. Malicious inbox rule removed; full-content mailbox audit enabled across C-suite. Finance team's wire approval process updated: all outbound transfers exceeding $10K require out-of-band phone verification with the requester before processing.

Detection Improvements
→ KQL analytic rule: Alert on inbox rule creation targeting an external domain within 60 minutes of an Impossible Travel event for the same UPN. Severity: High. Auto-opens P1 incident.
→ KQL analytic rule: Correlate sign-in events where session token age exceeds 4 hours and the source country differs from token-issuance country. Alert enriched with UEBA risk score via Sentinel fusion.
→ Logic App playbook: On confirmed AiTM indicator (phishing domain in sign-in referrer OR Tor exit node IP), auto-invoke revokeSignInSessions and page on-call analyst — zero manual steps for initial containment.
→ Continuous Access Evaluation (CAE) enforced in strict-location mode for all Executive security group accounts. Token lifetime reduced from 60 minutes to 10 minutes with IP-binding enabled.
